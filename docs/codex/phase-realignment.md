# Character Map Phase 재정렬안

기준일: 2026-04-09  
작성 목적: 문서상 Phase와 실제 코드 상태 불일치를 해소하기 위한 기준선 정리

## 재정렬 원칙

- 이미 구현된 코드는 완료로 인정한다.
- UI만 있고 서버가 없는 경우는 "부분 완료"로 구분한다.
- 다음 작업은 사용자 가치가 바로 올라가는 순서로 배치한다.

## 기존 vs 재정렬

기존 `current_phase.txt` 기준:

- Phase 1 완료
- Phase 2 완료
- Phase 3 진행 예정

실제 코드 기준 재평가:

- 작품 CRUD는 이미 구현되어 사실상 완료
- 인물/관계 UI는 존재하지만 백엔드 API 미구현
- 맵 UI는 기본 렌더링 가능하지만 고도화 미완료

## 재정렬된 Phase 정의

### Phase 1. 프로젝트 셋업 + DB 초기화 (완료)

포함 범위

- 디렉터리 구조
- 프론트/백엔드 개발 환경
- SQLite 연결/스키마 초기화

상태: 완료

### Phase 2. 인증 (완료)

포함 범위

- 로그인 화면
- JWT 발급/검증
- 인증 상태 기반 화면 분기

상태: 완료

### Phase 3. 작품 관리 CRUD (완료)

포함 범위

- `/api/works` CRUD
- 홈 작품 카드 그리드
- 타입/상태 필터
- 작품 추가/수정/삭제
- 작품 단위 export/import

상태: 완료

### Phase 4. 인물 CRUD (진행 예정)

포함 범위

- 인물 API 구현
- 인물 탭 UI와 서버 연결
- 즐겨찾기 토글 동작

상태: 다음 착수 대상

### Phase 5. 관계 CRUD (진행 예정)

포함 범위

- 관계 API 구현
- 관계 탭 UI와 서버 연결
- RelationForm ID 정합성 수정

상태: Phase 4 직후

### Phase 6. 시냅스 맵 통합 (부분 완료)

포함 범위

- Force Graph 기본 렌더링
- 인물/관계 실제 데이터 연동 안정화
- 상세/강조/필터 UX 보강

상태: 기본 구현 완료, 통합 고도화 필요

### Phase 7. 설정/마무리 (대기)

포함 범위

- 전체 export/import
- 데이터 초기화
- 운영 편의 기능

상태: 대기

### Phase 8. 배포 (대기)

포함 범위

- Nginx/PM2/EC2 실배포
- 운영 점검

상태: 대기

## 즉시 적용할 운영 기준

- `CURRENT PHASE`는 "Phase 4 — 인물 CRUD API 연결"로 갱신한다.
- 이후 세션 노트는 "완료/진행/다음" 3구역으로 고정 기록한다.
- 기능 완료 판단은 UI 존재가 아니라 "프론트-백엔드 실제 동작"으로 한다.
